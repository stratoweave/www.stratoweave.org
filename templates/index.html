StratoWeave

Intent-based network orchestration, built to close the loop.

Declarative transforms, YANG-driven abstraction, and native telemetry in one focused platform.

StratoWeave is a platform for robust network orchestration systems based on model-driven declarative transforms. It turns network intent into layered, typed automation, reacts to streaming telemetry, and ships as one binary with zero dependencies.

Core loop

From intent to live network state.

StratoWeave stays focused on one job: take northbound intent, apply layered transforms, push declarative configuration, and feed telemetry back into the next decision.

  1. 1. Capture intent Take service and policy input from NETCONF, RESTCONF, TMF640/641, and source-of-truth systems.
  2. 2. Layer transforms Break automation into tractable YANG-backed transforms instead of giant scripts.
  3. 3. Target real networks Render for Cisco XR, Juniper, Nokia, and other major network operating systems.
  4. 4. React to telemetry Consume streaming state and drive closed-loop automation from what the network reports.

How it works

StratoWeave uses transforms to move from service intent to verified state.

The platform is reactive end to end. It acts on configuration changes from NETCONF, RESTCONF, or TMF640/641 northbound APIs, or subscriptions on streaming telemetry from devices, and it uses typed YANG-backed layers to keep that logic understandable.

Northbound intent

Start from desired state.

Feed service intent, policies, and inventory into the system instead of hand-authoring device changes.

Layered transforms

Break complexity into typed stages.

Use YANG-derived types and layered transforms to move from service models down toward implementation detail.

Device output

Target multi-vendor networks.

Render configuration across Cisco XR, Juniper, Nokia, and other major network operating systems.

Telemetry feedback

Close the loop.

Collect streaming telemetry, compare live state to intent, and drive the next reaction from observed reality.
