Core selling points

The platform stays focused on network automation, not framework assembly.

The value proposition is narrow and practical: describe intent declaratively, layer transforms on typed models, close the loop with telemetry, and ship the whole thing as a small open platform that operators can actually run.

Completely intent-based

02

Layered YANG abstractions

Turn YANG models into typed layers so large automation problems break down into smaller tractable ones.

03

Telemetry-driven closed loop

Subscribe to streaming telemetry, observe live state, and react with the same transform system that drove the change.

How it works

StratoWeave uses transforms to move from service intent to verified state.

The platform is reactive end to end. It acts on configuration changes from NETCONF, RESTCONF, or TMF640/641 northbound APIs, or subscriptions on streaming telemetry from devices, and it uses typed YANG-backed layers to keep that logic understandable.

Northbound intent

Start from desired state.

Feed service intent, policies, and inventory into the system instead of hand-authoring device changes.

Layered transforms

Break complexity into typed stages.

Use YANG-derived types and layered transforms to move from service models down toward implementation detail.

Device output

Target multi-vendor networks.

Render configuration across Cisco XR, Juniper, Nokia, and other major network operating systems.

Telemetry feedback

Close the loop.

Collect streaming telemetry, compare live state to intent, and drive the next reaction from observed reality.

Deployment reality

Work with the network operating systems you already run.

StratoWeave is built for multi-vendor networks. The transform model lets you keep intent and abstractions stable while adapting the output for the platforms underneath.

Cisco XR Juniper Nokia Other major network OSs

Verify by telemetry Use live state to confirm that the rendered intent behaves the way the model expected.
